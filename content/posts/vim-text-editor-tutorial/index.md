Vim은 Vi 에디터의 확장판으로, Vi의 기능을 포함하면서 더 많은 기능을 제공하는 텍스트 편집기이다. Vim은 키보드만으로 조작할 수 있어 마우스 사용 없이도 빠르게 편집할 수 있는 장점이 있으며, 50년 가까운 역사를 통해 검증된 강력한 도구이다. 이번 포스트에서는 Vim의 역사부터 실전 활용법까지 포괄적으로 정리해 보겠다.

## Vim의 역사와 발전

### vi의 탄생

vi 에디터는 1976년 Bill Joy가 BSD Unix를 위해 개발한 텍스트 편집기로, 당시 느린 터미널 환경에서도 효율적으로 작동하도록 설계되었다. 모달 에디팅 개념을 도입하여 키보드만으로 빠른 편집이 가능하게 했고, 이는 현대 개발 환경에서도 여전히 유효한 철학이다.

### vim의 등장

vim은 1991년 Bram Moolenaar가 Amiga 컴퓨터용으로 개발한 Vi IMproved의 약자로, vi의 기능을 확장하고 개선한 버전이다. 문법 강조, 다중 실행 취소, 플러그인 지원, 그래픽 인터페이스 등 현대적인 기능을 추가했으며, 오픈 소스로 공개되어 전 세계 개발자들의 기여로 발전해 왔다.

### vim vs neovim

neovim은 2014년 vim의 코드베이스를 fork하여 시작된 프로젝트로, 비동기 플러그인 지원, 내장 LSP 클라이언트, Lua 스크립팅, 더 나은 기본 설정 등을 제공한다. vim은 전통적인 안정성과 호환성을 중시하며, neovim은 현대적인 개발 환경에 맞춘 혁신을 추구한다. 두 에디터 모두 활발히 개발되고 있고, 기본 사용법은 거의 동일하다.

### 왜 2025-2026년에도 vim이 사용되는지

vim은 거의 모든 Unix 계열 시스템에 기본 설치되어 있어 서버 환경에서 즉시 사용할 수 있고, SSH 접속 시 GUI 에디터보다 훨씬 가볍고 빠르며, 키보드 중심의 워크플로우는 익숙해지면 마우스보다 월등히 빠른 편집 속도를 제공한다. 또한 강력한 플러그인 생태계와 커스터마이징 가능성, 그리고 50년 가까이 축적된 커뮤니티 지식과 리소스가 vim의 지속적인 인기를 뒷받침한다.

## 모달 에디팅의 철학

### 모달 에디팅이란

모달 에디팅은 편집기의 동작을 여러 모드로 분리하여 같은 키가 모드에 따라 다른 기능을 수행하는 방식이다. 일반 에디터에서는 문자 키를 누르면 항상 해당 문자가 입력되지만, vim에서는 Normal 모드에서는 명령을 실행하고 Insert 모드에서만 텍스트를 입력한다. 이러한 분리는 처음에는 혼란스럽지만, 익숙해지면 매우 강력한 편집 도구가 된다.

### 왜 모달 에디팅이 강력한지

모달 에디팅은 키보드의 모든 키를 명령어로 활용할 수 있게 하여 Ctrl이나 Alt 같은 수식 키 없이도 수백 가지 명령을 간결하게 사용할 수 있고, 텍스트 입력과 편집을 명확히 분리하여 손목 부담을 줄이며, 명령어들을 조합하여 복잡한 편집 작업을 간단히 표현할 수 있다. 또한 반복 작업에 최적화되어 있어 매크로나 점 명령어로 생산성을 극대화할 수 있다.

### 학습 곡선과 장기적 생산성

vim의 초기 학습 곡선은 가파르지만, 기본 명령어 20-30개만 익히면 일상적인 편집이 가능하고, 몇 주 정도 사용하면 기존 에디터와 비슷한 속도로 작업할 수 있으며, 몇 달 후에는 기존 에디터보다 훨씬 빠른 편집 속도를 경험하게 된다. 투자한 시간은 수십 년간 사용할 수 있는 기술로 돌아오며, vim 키 바인딩은 많은 도구에서 지원되어 다양한 환경에서 활용할 수 있다.

## Vim 모드

Vim은 크게 4가지 모드로 나뉜다.

1. Normal 모드: 기본 모드로, 텍스트를 편집하거나 명령을 입력할 수 있는 모드이다.
2. Insert 모드: 텍스트를 입력할 수 있는 모드로, 텍스트를 입력하거나 수정할 수 있다.
3. Visual 모드: 텍스트를 선택할 수 있는 모드로, 선택한 텍스트를 복사하거나 삭제할 수 있다.
4. Command 모드: 콜론으로 시작하는 명령어를 입력하여 파일 저장, 검색, 치환 등을 수행하는 모드이다.

## 모드별 상세 명령어

### Normal 모드

Vim을 실행하면 기본적으로 Normal 모드로 시작된다. Normal 모드는 vim의 핵심이며, 대부분의 시간을 이 모드에서 보내게 된다.

#### 이동 명령어

-   `h`, `j`, `k`, `l`: 왼쪽, 아래, 위, 오른쪽으로 한 칸 이동
-   `w`: 다음 단어의 시작으로 이동
-   `b`: 이전 단어의 시작으로 이동
-   `e`: 현재 단어의 끝으로 이동
-   `gg`: 파일의 첫 줄로 이동
-   `G`: 파일의 마지막 줄로 이동
-   `0`: 줄의 맨 앞으로 이동
-   `$`: 줄의 맨 뒤로 이동
-   `f{문자}`: 현재 줄에서 해당 문자로 앞으로 이동
-   `F{문자}`: 현재 줄에서 해당 문자로 뒤로 이동
-   `t{문자}`: 해당 문자 직전까지 앞으로 이동
-   `T{문자}`: 해당 문자 직후까지 뒤로 이동

#### 편집 명령어

-   `x`: 커서 위치의 문자 삭제
-   `X`: 커서 앞의 문자 삭제
-   `r{문자}`: 커서 위치의 문자를 다른 문자로 교체
-   `R`: Replace 모드로 전환하여 연속 교체
-   `c{동작}`: 지정한 범위를 삭제하고 Insert 모드로 전환
-   `C`: 커서부터 줄 끝까지 삭제하고 Insert 모드로 전환
-   `s`: 커서 위치의 문자를 삭제하고 Insert 모드로 전환
-   `S`: 현재 줄 전체를 삭제하고 Insert 모드로 전환
-   `dd`: 현재 줄 삭제
-   `yy`: 현재 줄 복사
-   `p`: 커서 뒤에 붙여넣기
-   `P`: 커서 앞에 붙여넣기
-   `u`: 실행 취소
-   `Ctrl + r`: 실행 취소 취소

### Insert 모드

Insert 모드는 실제로 텍스트를 입력하는 모드로, 다양한 방법으로 진입할 수 있으며 각각의 시작 위치가 다르다.

-   `i`: 커서 위치에서 입력 시작
-   `a`: 커서 다음 위치에서 입력 시작
-   `o`: 현재 줄 아래에 새 줄을 만들고 입력 시작
-   `O`: 현재 줄 위에 새 줄을 만들고 입력 시작
-   `I`: 줄의 맨 앞에서 입력 시작
-   `A`: 줄의 맨 뒤에서 입력 시작

Insert 모드에서는 `Esc` 키를 누르면 Normal 모드로 전환된다. 많은 vim 사용자들은 `Esc` 키가 멀어서 `Caps Lock`을 `Esc`로 매핑하거나 `Ctrl + [` 단축키를 사용한다.

### Visual 모드

Visual 모드는 텍스트를 시각적으로 선택하여 편집할 수 있는 모드로, 세 가지 타입이 있다.

-   `v`: 문자 단위 Visual 모드로 전환
-   `V`: 줄 단위 Visual 모드로 전환
-   `Ctrl + v`: 블록 단위 Visual 모드로 전환하여 직사각형 영역 선택

Visual 모드에서 선택 후 사용할 수 있는 명령어는 다음과 같다.

-   `d`: 선택한 텍스트 삭제
-   `y`: 선택한 텍스트 복사
-   `c`: 선택한 텍스트 삭제하고 Insert 모드로 전환
-   `>`: 선택한 텍스트 들여쓰기
-   `<`: 선택한 텍스트 내어쓰기

### Command 모드

Command 모드는 콜론으로 시작하는 명령어를 입력하여 파일 작업, 설정 변경, 복잡한 편집 작업을 수행한다.

-   `:w`: 파일 저장
-   `:q`: vim 종료
-   `:wq` 또는 `:x`: 저장 후 종료
-   `:q!`: 저장하지 않고 강제 종료
-   `:e {파일명}`: 다른 파일 열기
-   `:set number`: 줄 번호 표시
-   `:set tabstop=4`: 탭 크기를 4로 설정
-   `:help {주제}`: 도움말 보기

## 검색과 치환

### 검색 기능

vim의 검색 기능은 매우 강력하고 정규표현식을 지원한다.

-   `/패턴`: 현재 위치에서 아래로 검색
-   `?패턴`: 현재 위치에서 위로 검색
-   `n`: 같은 방향으로 다음 검색 결과로 이동
-   `N`: 반대 방향으로 이전 검색 결과로 이동
-   `*`: 커서 위치의 단어를 아래로 검색
-   `#`: 커서 위치의 단어를 위로 검색

### 치환 명령어

치환 명령어는 텍스트를 일괄적으로 변경할 때 사용하며, 정규표현식과 함께 사용하면 매우 강력하다.

-   `:s/old/new/`: 현재 줄에서 첫 번째 old를 new로 치환
-   `:s/old/new/g`: 현재 줄에서 모든 old를 new로 치환
-   `:%s/old/new/g`: 파일 전체에서 모든 old를 new로 치환
-   `:%s/old/new/gc`: 파일 전체에서 확인하며 치환
-   `:5,10s/old/new/g`: 5줄부터 10줄까지 치환

## 유용한 플러그인

### vim-plug 플러그인 매니저

vim-plug는 가장 인기 있는 vim 플러그인 매니저로, 플러그인 설치와 업데이트를 간편하게 관리할 수 있다. GitHub에서 한 줄 명령어로 설치할 수 있고, vimrc에 플러그인 목록을 작성한 후 `:PlugInstall` 명령으로 일괄 설치할 수 있다.

### NERDTree

NERDTree는 파일 탐색기 플러그인으로, 사이드바에 트리 구조로 파일 시스템을 표시하고 파일 생성, 삭제, 이동 등을 GUI처럼 수행할 수 있다. `:NERDTreeToggle` 명령으로 토글할 수 있으며, 많은 개발자들이 필수로 사용하는 플러그인이다.

### fzf

fzf는 퍼지 파인더 플러그인으로, 파일명이나 버퍼, 태그를 빠르게 검색할 수 있게 해준다. 부분 문자열 매칭으로 파일을 찾을 수 있어 프로젝트에서 원하는 파일을 즉시 열 수 있고, 명령어 히스토리 검색도 지원한다.

### coc.nvim

coc.nvim은 현대적인 자동완성과 LSP 클라이언트를 제공하는 플러그인으로, VSCode와 유사한 수준의 인텔리센스를 vim에서 사용할 수 있게 한다. 코드 자동완성, 정의로 이동, 참조 찾기, 에러 표시 등 현대적인 IDE 기능을 제공하며, 다양한 언어를 지원한다.

### vim-airline

vim-airline은 상태바 플러그인으로, 화면 하단에 현재 모드, 파일명, 줄 번호, Git 브랜치 등 유용한 정보를 예쁘게 표시한다. 다양한 테마를 지원하고 다른 플러그인들과 통합되어 추가 정보를 표시할 수 있다.

## vimrc 설정 예제

vimrc는 vim의 설정 파일로, 홈 디렉토리의 `.vimrc` 파일에 작성한다. 다음은 기본적인 설정 예제이다.

```vim
" 기본 설정
set number              " 줄 번호 표시
set relativenumber      " 상대 줄 번호 표시
syntax on               " 문법 강조
set tabstop=4           " 탭 크기 4
set shiftwidth=4        " 들여쓰기 크기 4
set expandtab           " 탭을 스페이스로 변환
set autoindent          " 자동 들여쓰기
set smartindent         " 스마트 들여쓰기
set hlsearch            " 검색 결과 강조
set incsearch           " 점진적 검색
set ignorecase          " 대소문자 구분 안 함
set smartcase           " 대문자 입력 시 구분함

" 키 매핑
let mapleader = " "     " 리더 키를 스페이스로 설정
nnoremap <leader>w :w<CR>                " 스페이스+w로 저장
nnoremap <leader>q :q<CR>                " 스페이스+q로 종료
nnoremap <C-h> <C-w>h                    " Ctrl+h로 왼쪽 창 이동
nnoremap <C-l> <C-w>l                    " Ctrl+l로 오른쪽 창 이동
```

## 실전 팁과 단축키 조합

### 매크로 사용

매크로는 반복 작업을 자동화하는 강력한 기능으로, `q{레지스터}`로 녹화를 시작하고 작업을 수행한 후 `q`로 녹화를 종료한다. `@{레지스터}`로 매크로를 재생할 수 있고, `@@`로 마지막 매크로를 반복 실행할 수 있으며, `10@a`처럼 숫자와 함께 사용하면 여러 번 실행할 수 있다.

### 여러 파일 편집

vim에서는 여러 파일을 동시에 열어 편집할 수 있다.

-   `:e 파일명`: 새 파일 열기
-   `:bn`: 다음 버퍼로 이동
-   `:bp`: 이전 버퍼로 이동
-   `:ls`: 열린 버퍼 목록 보기
-   `:b 번호`: 해당 번호의 버퍼로 이동

### 분할 화면

화면을 분할하여 여러 파일을 동시에 볼 수 있다.

-   `:split` 또는 `:sp`: 가로로 화면 분할
-   `:vsplit` 또는 `:vs`: 세로로 화면 분할
-   `Ctrl + w + h/j/k/l`: 분할된 창 사이 이동
-   `Ctrl + w + =`: 모든 창 크기 균등하게 조정

### 점 명령어로 반복

점(`.`) 명령어는 마지막 변경 작업을 반복하는 매우 유용한 기능이다. 예를 들어 `ciw`로 단어를 변경하고 새 단어를 입력한 후, 다른 위치에서 `.`을 누르면 같은 작업이 반복된다. 이는 매크로보다 간단한 반복 작업에 적합하다.

## vimtutor 활용

vimtutor는 vim에 내장된 대화형 튜토리얼로, 터미널에서 `vimtutor` 명령어를 입력하면 실행된다. 약 30분 정도 소요되며, vim의 기본 명령어를 직접 실습하면서 배울 수 있어 초보자에게 가장 좋은 학습 방법이다. 한글 버전도 지원되며, 여러 번 반복하면서 손에 익히는 것이 중요하다.

## 정리

Vim은 50년 가까운 역사를 가진 강력한 텍스트 편집기로, 모달 에디팅이라는 독특한 철학을 통해 키보드만으로 매우 빠른 편집이 가능하다. 초기 학습 곡선은 가파르지만 익숙해지면 다른 어떤 에디터보다 효율적이고, 서버 환경에서의 필수 도구이며, 플러그인과 설정을 통해 현대적인 IDE 수준의 기능을 갖출 수 있다. Vim의 키 바인딩은 많은 도구에서 지원되므로 한 번 배우면 평생 사용할 수 있는 투자 가치가 높은 기술이다. vimtutor로 시작하여 매일 조금씩 사용하다 보면 자연스럽게 손에 익게 되고, 언젠가는 vim 없이는 편집할 수 없게 되는 경험을 하게 될 것이다.
